# Sprint Retro - Sprint #3

**Date:** 2026-02-20

## What Went Well
- Rapid iteration on visual tweaks based on stakeholder screenshots.
- Edge fade implementations for hero and trusted logos validated quickly.

## What Slowed Us Down
- Frequent mid-task visual adjustments (typography and spacing) caused rework.
- Portfolio sizing/edge behavior required multiple passes to match references.

## Action Items
- Lock typography targets with explicit pixel values before implementation.
- Capture reference-based measurements early (card widths, paddings).
- For visual tasks, collect required breakpoints and exact values up front.
